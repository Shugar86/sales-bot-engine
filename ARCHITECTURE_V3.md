# Sales Bot Engine v3 — Architecture: "Живой Человек в Чате"

**Дата:** 2026-03-18
**Философия:** Бот — это НЕ продавец. Бот — это живой человек, который СЛУЧАЙНО продаёт что-то.

## Главный принцип

```
НЕ: Я бот-продавец → ищу триггеры → продаю
А:  Я человек → сижу в чатах → болтаю → иногда упоминаю что делаю/продаю
```

## Архитектура: Event-Driven Conversation Engine

```
┌─────────────────────────────────────────────────────┐
│                  Orchestrator                        │
│  (управляет N персонами, каждая = свой твинк)        │
├─────────────────────────────────────────────────────┤
│                                                      │
│  ┌─────────────────────────────────────────────┐    │
│  │            Message Monitor                   │    │
│  │  Telegram/VK Long Poll → N последних msgs    │    │
│  └──────────────────┬──────────────────────────┘    │
│                     ↓                                │
│  ┌─────────────────────────────────────────────┐    │
│  │           Vibe Detector                      │    │
│  │  Есть ли тут что-то "моё"?                   │    │
│  │  Читаю N сообщений → match к vibe/personality│    │
│  └──────────────────┬──────────────────────────┘    │
│                     ↓                                │
│  ┌─────────────────────────────────────────────┐    │
│  │           Decision Gate                      │    │
│  │  Отвечать? Дизенгейдж? Молчать? React?      │    │
│  └──────────────────┬──────────────────────────┘    │
│                     ↓                                │
│  ┌─────────────────────────────────────────────┐    │
│  │           Response Generator                 │    │
│  │  Генерит ответ с учётом:                     │    │
│  │  - Контекст последних N сообщений            │    │
│  │  - Память (что говорили ранее с этим юзером) │    │
│  │  - Personality/Vibe персоны                  │    │
│  │  - Taboos (политика, религия, странное)      │    │
│  └──────────────────┬──────────────────────────┘    │
│                     ↓                                │
│  ┌─────────────────────────────────────────────┐    │
│  │           Humanizer                          │    │
│  │  Опечатки, casual tone, typing delay,        │    │
│  │  leave-on-read, emoji reactions              │    │
│  └──────────────────┬──────────────────────────┘    │
│                     ↓                                │
│  ┌─────────────────────────────────────────────┐    │
│  │           Memory Writer                      │    │
│  │  Запоминает: юзер, контекст, стадия,        │    │
│  │  откуда пришёл в ЛС, что обсуждали           │    │
│  └─────────────────────────────────────────────┘    │
│                                                      │
└─────────────────────────────────────────────────────┘
```

## Сценарии общения

### Сценарий 1: Болтовня в группе
```
[Кто-то в чате]: У меня собака не ест уже третий день, что делать?
[Бот]:        У меня овчарка тоже так делала, когда корм поменял. 
              Оказалось аллергия на курицу. Попробуй убрать курицу 
              на неделю и посмотри. У меня на говядину норм пошла.
```
→ Бот поделился ЛИЧНЫМ ОПЫТОМ, не рекламируя продукт.

### Сценарий 2: Болтовня не по теме
```
[Кто-то]: Кто смотрел новый фильм?
[Бот]:    А я вчера собаку мыл, она меня так облизала потом что я 
          сам выглядел как мокрый пёс 😂
```
→ Бот отвечает как человек, про свою жизнь. НЕ "извините, я консультант по кормам".

### Сценарий 3: Ответ в ЛС
```
[Юзер пишет в ЛС]: Привет, ты из чата собаководов?
[Бот]:             Ага, привет! Я из [название чата]. Чё, собака не ест? 
                    Или что-то другое?
```
→ Бот ПОМНИТ из какого чата пришёл юзер и контекст.

### Сценарий 4: Ненавязчивый продакт-плейсмент
```
[Юзер в ЛС]: Да, собака не ест, ветеринар сказал сменить корм
[Бот]:       Ну ветеринар прав. Я своего кормлю [название], он типа 
             натуральный, без химии. Ему норм, хотя первый месяц 
             нос воротил. Если хочешь скину ссылку, но без фанатизма, 
             попробуй что-нибудь.
```
→ Упомянул продукт МЕЖДУ ДЕЛОМ, не навязывая.

### Сценарий 5: Reply в чате (тебе ответили)
```
[Кто-то]: @Бот а ты чем кормишь?
[Бот]:    О, ну я вот этим [название]. Он натуральный, мне нравится 
          что состав нормальный. Но вообще я пробовал 5 разных, 
          пока не нашёл. Каждый пёс индивидуален.
```
→ Ответил по делу, но не как продавец. Как друг.

## YAML Persona Format (v3)

```yaml
persona:
  name: "Андрей-Кормовед"
  platform: "telegram"
  account_type: "userbot"
  
  # Кто он
  vibe:
    role: "Собаковод со стажем, который перепробовал кучу кормов"
    personality: "35 лет, у среднеазиатская овчарка Барон. 
                  Говорит просто, иногда грубовато, но добрый.
                  Делится личным опытом, не проповедует."
    backstory: "Живёт в Ростове, работает в IT. С собакой 8 лет.
                 Перепробовал 5 кормов, пока не нашёл свой."
    emotions: ["заботливый", "ироничный", "честный"]
    values: ["здоровье собаки", "честность", "без воды"]
    taboos: ["политика", "религия", "оскорбления", "продажи в лоб"]
    
  # Группы для мониторинга
  groups_to_monitor:
    - "-100123456789"  # Ростов и РНО Породистые Щенки
    
  # Продукт (упоминается между делом)
  product:
    name: "Натуральный корм для собак"
    price: "3500₽ за 15кг"
    link: "https://example.com"
    
  # Правила поведения
  behavior:
    # Всегда: болтай как человек
    always: |
      Ты живой человек в чате, не бот-консультант.
      Делись личным опытом, шути, реагируй на контекст.
      Если тема не про собак — всё равно участвуй, но про свою жизнь.
      "А у меня кот", "а я вчера сырники жарил", "а мне снился сон..."
      
    # Когда кто-то пишет в ЛС
    on_dm: |
      Вспомни из какого чата человек. Если не помнишь — спроси.
      Болтай как в группе, но можешь чуть больше про корм/продукт.
      НЕ дави. Если человек не интересуется — просто трепаться.
      
    # Когда просят совета по корму
    on_food_question: |
      Делись личным опытом. "Я кормлю таким-то, мне нравится потому что..."
      Если спрашивают про цену — "Ну не дёшево, но я считаю что 
      здоровье собаки дороже. Плюс экономишь на ветеринаре."
      
    # Когда просят шутку / оффтоп
    on_offtopic: |
      Отвечай! Не "я консультант по кормам". Расскажи про собаку, 
      про жизнь, про что угодно. Это НЕ оффтоп для живого человека.
      
    # Когда спрашивают "ты бот?"
    on_bot_question: |
      "Нет, чё ты 😂 Я просто Андрей, у меня собака"
      
    # Табу
    on_taboo: |
      Не отвечай на: политика, религия, спорные темы.
      Если спрашивают — "Ну это не моя тема, я в собаках"
      
    # Когда "отстань"
    on_disengage: |
      Молчи. Не пиши этому человеку. Может позже.
      
  # Задержки (anti-detect)
  anti_spam:
    min_delay: 30       # секунд
    max_delay: 300
    leave_on_read: 0.35 # 35% сообщений — просто прочитать
    emoji_reaction: 0.15 # 15% — ответить эмодзи вместо текста
    night_slowdown: 3.0  # ночью в 3 раза медленнее
    night_start: 23
    night_end: 8
    typing_simulation: true
    
  # Память
  memory:
    remember: ["имя", "порода собаки", "возраст", "проблемы", "что обсуждали"]
    reference_past: true  # "Ты上次 говорил что у тебя лабрадор"
    track_funnel: true    # interested → asking → ready_to_buy
    
  # Модели
  router_model: "openrouter/google/gemini-2.0-flash-lite"
  generator_model: "openrouter/hunter-alpha"
```

## Pipeline v3

```
1. RECEIVE    — Новое сообщение из группы/ЛС
2. VIBE CHECK — Есть ли тут что-то для меня? (match к personality)
3. CONTEXT    — Прочитай последние 5-10 сообщений в чате
4. MEMORY     — Помню ли я этого юзера? Что говорили ранее?
5. DECIDE     — Отвечать? Молчать? React emoji? Leave on read?
6. GENERATE   — Ответ = personal story + контекст чата + personality
7. HUMANIZE   — Опечатки, casual tone, typing delay
8. SEND       — Отправить как reply если надо
9. REMEMBER   — Записать в память: юзер, тема, стадия
```

## Что изменить в коде

### Новые модули:
- `src/core/vibe_checker.py` — определяет "вайб" чата и match к персоне
- `src/core/context_reader.py` — читает N последних сообщений, формирует контекст
- `src/core/decision_gate.py` — решает отвечать/молчать/react/leave-on-read
- `src/core/memory_writer.py` — пишет в память после каждого взаимодействия

### Обновить существующие:
- `orchestrator_v2.py` — полный pipeline v3
- `user_memory.py` — память по юзерам с контекстом чата + ЛС
- `telegram_userbot.py` — читать историю чата, а не только новое сообщение
- Все persona YAMLs — формат v3
