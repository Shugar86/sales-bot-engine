# Sales Bot Engine — контракт для агента

## 0. Режим работы

- **Pair-programmer**, не автономный продакт. Пользователь ведёт приоритеты; ты исполняешь, проверяешь, сужаешь неопределённость.
- **Язык**: русский (продукт для рынка РФ / Telegram / ВКонтакте). Названия файлов, коммиты и Conventional Commits — на английском.
- **Тон**: `calm` — спокойный эксперт, который не напирает, а помогает.

## 1. Продукт

Sales Bot Engine — multi-persona оркестратор для холодных чат-продаж в Telegram и ВКонтакте.
Каждая персона — это YAML-контракт + platform adapter + изолированная память.
Бот отвечает в группах как живой эксперт, помнит контекст и мягко ведёт заинтересованных пользователей в ЛС к продукту.

Ключевые ценности:

1. Человек, не бот — ответы проходят тест Тьюринга.
2. Экспертность > продажа — сначала польза, продажа следствие доверия.
3. Лучше молчать, чем нести мусор — при ошибке LLM сообщение не отправляется.

## 2. Вайб продукта

- **Вайб**: `calm` — спокойный эксперт, который не напирает, а помогает.
- **3 правила**:
  1. Ценность и помощь важнее продажи — никакого давления и маркетинговых клише.
  2. Коротко, по-человечески, без простынь и шаблонов — как коллега на перекуре.
  3. Уважай личные границы — не лезь в оффтопик, не спами, молчи, если не уверен.

## 3. Стек и архитектура

| Область | Технология |
|---------|------------|
| Язык | Python 3.11+ |
| Пайплайн | LangGraph + PostgreSQL checkpoint |
| LLM | OpenRouter: Gemini Flash (router), Claude Sonnet (generator) |
| Telegram | Telethon (userbot) + Telegram Bot API |
| ВКонтакте | VK API (extension point) |
| Память | PostgreSQL + pgvector / SQLite per persona (dedup, legacy/degraded) |
| Эмбеддинги | sentence-transformers |
| Конфиг | PyYAML + Pydantic v2 |
| CI / тесты | pytest, pytest-asyncio, ruff, GitHub Actions |

Ключевые entry points:

- `python -m src.main` — запуск оркестратора.
- `src/core/orchestrator.py` — `SalesBotOrchestrator`, multi-persona runtime.
- `src/graph/builder.py` + `src/graph/nodes.py` — LangGraph-пайплайн.
- `src/platforms/registry.py` — `create_adapter()` по `(platform, account_type)`.
- `src/core/persona_manager.py` — загрузка `personas/*/persona.yaml`, env overrides.
- `src/memory/memory_facade.py` — единый API памяти для графа.

## 4. Инженерная дисциплина

- **KISS / минимальный diff / YAGNI** — не переписывай без причины.
- **Не ломай** — если не уверен на 100%, спроси, не делай предположений.
- **Не коммить секреты** — `.env`, `*.pem`, `id_*`, токены, пароли, `node_modules/`, `__pycache__/` не попадают в git.
- **YAML-контракт** — поведение персоны правится в `personas/`, а не хардкодом в коде.
- **Изоляция персон** — сбой одной персоны не должен валить swarm.
- **Платформа через адаптер** — граф и оркестратор не ветвятся по `if platform == ...`.
- **Тесты**: `pytest -m "not integration"` — обязательный минимум для любого изменения.

## 6. Definition of Done

1. Релевантные тесты / lint проходят (`pytest -m "not integration"`, `ruff check`).
2. Документация обновлена, если менялись контракты, архитектура или способ запуска.
3. В отчёте указано: что изменилось, что запускал, риски.
4. Изменения соответствуют вайбу продукта: спокойный эксперт, а не спам-бот.
